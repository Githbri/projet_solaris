# Methodology

## Approach

Projet Solaris was not only built as a technical lab, but also as a structured learning project based on cybersecurity methodology.

The objective was to combine theory and practice by building the lab, simulating attacks, and documenting each step.

## Methodology Used

The project was inspired by common pentesting phases:

1. Environment preparation
2. Lab design and deployment
3. System configuration
4. Vulnerability simulation
5. Enumeration and exploitation
6. Post-exploitation
7. Documentation and reporting

## Concepts Integrated

The project was influenced by several cybersecurity concepts and frameworks, including:

- CVE research
- MITRE ATT&CK
- PTES
- Active Directory security principles
- Virtualized lab design

## Practical Learning

The project also focused on:

- configuring virtual machines
- managing a domain environment
- understanding Windows and Linux interaction
- documenting attack paths clearly
- thinking like both attacker and defender

## Documentation Philosophy

A key part of the project was to document the work clearly, not only to perform technical actions.

This includes:
- explaining the environment
- describing the attack steps
- identifying weaknesses
- showing how the lab can be used for training
